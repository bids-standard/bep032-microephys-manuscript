\begin{abstract}
	The Brain Imaging Data Structure (BIDS) has facilitated data sharing and tool development for neuroscience data so far primarily in the human research domain. We present an extension to BIDS for animal microelectrode electrophysiology, addressing the distinct requirements of intracellular and extracellular recordings.
	This extension introduces two new data types: \texttt{icephys} for intracellular and \texttt{ecephys} for extracellular recordings, supporting diverse recording modalities from patch-clamp to high-density silicon probes.
	Building on existing BIDS principles and prior electrophysiology extensions for EEG and iEEG, we specify metadata for probes, electrodes, and channels, paying particular attention to metadata required for spike sorting analysis.
	The extension adopts NWB (Neurodata Without Borders) and NIX (Neuroscience Information Exchange) as standards for storing neural recordings, ensuring comprehensive metadata capture while maintaining compatibility with existing analysis ecosystems.
	We provide example datasets covering common use cases and discuss integration with established tools.
	This standardization enables reproducible analysis pipelines, facilitates data sharing through data repositories like DANDI, G-Node and EBRAINS, and bridges different scales from cellular to systems neuroscience.
\end{abstract}

\section*{Introduction}

Microelectrode electrophysiology encompasses techniques for recording electrical activity on spatial scales smaller than one millimeter, such as the membrane potential of individual neurons and local field potentials.
These measurements provide crucial insights into neural computation at high spatial and temporal resolution, often with the ability to track individual neuronal action potentials.
Recent technological advances, including high-density silicon probes such as Neuropixels~\citep{jun_fully_2017, Steinmetz2021}, have dramatically increased the volume of the acquired data. 
Single probes are now capable of recording extracellularly from hundreds of neurons simultaneously generating more than $\approx 80$~GB/h per probe~\citealp{Steinmetz2021}.
Intracellular techniques, such as patch clamp recordings~\citep{sakmann_patch_1984}, are a complementary class of experimental methods that provide direct access to cellular membrane dynamics, enabling detailed characterization of synaptic transmission and intrinsic neuronal properties.
While the amount of acquired data for intracellular techniques is typically less than those estimated above, they share a similarly high degree of heterogeneity from the underlying experimental diversity.
These two classes of methods are used primarily in animal research and generate data of increasing complexity that demand a standardized organization.

A key challenge in the microelectrode electrophysiology community has been the proliferation of proprietary file formats.
Different acquisition systems, such as Blackrock, Plexon, TDT, Open Ephys, and SpikeGLX, each produce data in their own formats.
While to some extent this fragmentation is rooted in a data recorder's need to optimize the file format for write efficiency given a certain hardware design, it creates barriers that complicate data sharing, tool interoperability, and long-term archiving of data. 
Dedicated readers must be maintained for each format. For example, the Neo ~\citep[\rrid{RRID:SCR\_000634}; ][]{garcia_neo_2014} library for data handling in electrophysiology currently supports over 40 different interfaces to different file formats. Despite the availability of such software tools, undocumented specifications, unmaintained programmatic interfaces, and language lock-in risk rendering data inaccessible.

Two major community efforts have emerged to address this challenge at the file format level for individual recording sessions.
Neurodata Without Borders (NWB, \rrid{RRID:SCR\_015242})~\citep{rubel_neurodata_2022, teeters_neurodata_2015} is an open data standard for neurophysiology developed as part of the National Institutes of Health (NIH) BRAIN Initiative by a community of researchers over the last 10 years.
NWB uses HDF5 or Zarr as its storage backend and provides a comprehensive schema for storing electrophysiology data alongside stimulus information, behavioral data, and metadata.
The Neuroscience Information Exchange Format (NIX, \rrid{RRID:SCR\_016196})~\citep{adrian_file_2014} is an open data model for annotated self-descriptive scientific datasets that provides an HDF5 file backend for the Neo data model.
Both standards are widely adopted in the neuroscience community, are supported by widely-used analysis toolkits such as SpikeInterface (\rrid{RRID:SCR\_021150})~\citep{buccino_spikeinterface_2020}, and have been endorsed by the International Neuroinformatics Coordinating Facility (INCF) as community standards for neuroscience data storage.
NWB and NIX standardize the representation of data at the single-file level, which is typically used to store data of a single recording session. These file formats allow tool developers to build analysis pipelines that work across laboratories~\citep{denker_collaborative_2018, gutzen_2014, viejo_pynapple_2023, magland_neurosift_2024}.
However, neither standard provides organizational guidance at the study level for organizing collections of NWB or NIX files that represent recordings from many sessions, subjects, and experimental conditions within a study.
Researchers must still make \textit{ad hoc} decisions about directory structures, naming conventions, and the placement of study-level metadata, such as participant information and experimental protocols. In addition, some of the basic metadata describing the specific electrophysiology setting of the study, such as electrode configurations, remain difficult to access as part of the included binary files.

The Brain Imaging Data Structure (BIDS, \rrid{RRID:SCR\_016124})~\citep{gorgolewski_brain_2016} is a community-driven standard for organizing and sharing neuroscience data. Originally developed for MRI, BIDS has been extended through community-driven proposals to additional modalities including MEG~\citep{niso_meg-bids_2018}, EEG~\citep{pernet_eeg-bids_2019}, iEEG~\citep{holdgraf_ieeg-bids_2019}, PET, and microscopy.
With over a thousand public datasets on OpenNeuro and endorsement by the INCF, BIDS has become a widely adopted framework for FAIR neuroscience data~\citep{wilkinson_fair_2016}.

In a BIDS dataset, all data from each subject are organized hierarchically in subject-specific directories, with optional session subdirectories containing modality-specific folders for the data files themselves.
Files follow a structured naming scheme that encodes relevant metadata such as subject, session, and task.
Additional metadata are encoded in sidecar metadata files with formal schemas. Specifically, JSON sidecar files provide detailed metadata for each data file, while TSV files capture tabular information, such as participant demographics and channel properties.
A \texttt{dataset\_description.json} file describes the dataset as a whole, and JSON metadata at higher directory levels are inherited by lower levels unless overridden.

Two existing EEG and iEEG extensions for BIDS~\citep{pernet_eeg-bids_2019, holdgraf_ieeg-bids_2019} established patterns for organizing time-series neural data for meso- and macro-scale electrophysiology experiments.
Building on these two extensions, we present an extension for microelectrode electrophysiology that bridges the gap between session-level data standardization (NWB/NIX) and study-level organization (BIDS), combining the strengths of both approaches.
BIDS provides the organizational framework for datasets containing multiple subjects and sessions, consistent with existing modalities, while NWB and NIX handle the complex internal structure of electrophysiology recordings within each session.
This complementary relationship mirrors how BIDS for neuroimaging organizes NIfTI files.
BIDS does not replace standardized file formats, but provides the organizational context around them.

The design of this enhancement proposal reflects the unique demands of microelectrode electrophysiology while preserving compatibility with existing (BIDS) conventions and BIDS extensions whenever possible. In particular, this effort addresses the following key challenges:
\begin{enumerate}[noitemsep]
	\item Electrode scales that span orders of magnitude (submicrometer tips to millimeter arrays).
	\item Diverse target species with different anatomical brain reference spaces and atlases.
	\item Large data volumes, which can exceed $100~GB$ for an individual session.
	\item Diverse probe geometries, requiring specialized coordinate systems.
	\item Spike sorting as an essential pre-processing step, requiring specific metadata.
\end{enumerate}

Here, we present the extension of BIDS to microelectrode electrophysiology with a focus on animal models.
We define two datatypes, ecephys and icephys, introduce a three-level metadata model spanning probes, electrodes, and channels, and specify how physical, stereotaxic, and atlas-based coordinate information should be represented. Together, these additions provide a project-level organizational standard for collections of NWB and NIX files across subjects and sessions.

\subsection*{Standardization Landscape}

The MINI checklist~\citep{gibson_minimum_2009}, developed by the CARMEN consortium, was among the first community efforts to define what metadata should accompany electrophysiology recordings, covering subject, electrode, recording location, and stimulus information, but explicitly left data format and organization out of scope.
odML (\rrid{RRID:SCR\_001376})~\citep{grewe_bottom-up_2011} addressed how to structure such metadata through flexible key-value annotation and was adopted in practice for multi-electrode array datasets~\citep{zehl_handling_2016}.
NWB~\citep{teeters_neurodata_2015} and NIX~\citep{adrian_file_2014} then standardized the data format itself, embedding metadata within self-describing files, while Neo~\citep{garcia_neo_2014} provided the programmatic I/O layer that bridges proprietary acquisition formats to open standards.

At the project level, large-scale collaborations developed their own organizational conventions to organize collections of files from many recording sessions and modalities. For example, the Allen Brain Observatory~\citep{de_vries_sharing_2023} adopted internal data management practices and the International Brain Laboratory designed the Open Neurophysiology Environment (ONE)~\citep{one2024} to coordinate Neuropixels recordings across dozens of laboratories.
BEP032 described here occupies this project-level layer, providing a community standard that builds on the mature ecosystems from these earlier initiatives and that could serve as a common baseline for such collaborative efforts.

\subsection*{Creating a BIDS-compliant microelectrode electrophysiology dataset}

A typical workflow for creating a BIDS-compliant microelectrode electrophysiology  (microephys) dataset consists of four steps:
\begin{enumerate}[noitemsep]
	\item \textbf{Convert raw data.}
	      Convert recordings from the acquisition system's proprietary format to NWB or NIX\@.
	      The Neo library~\citep{garcia_neo_2014} provides readers for dozens of proprietary acquisition formats and serves as the foundation for conversion tools; NeuroConv~\citep{mayorquin_neuroconv_2025} builds on Neo to automate conversion to NWB, while Neo also supports writing NIX files directly.
	\item \textbf{Organize into BIDS.}
	      Arrange the converted files into a BIDS directory structure and generate sidecar metadata files.
	      nwb2bids \citep{nwb2bids_zenodo} extracts metadata from NWB files to populate BIDS sidecar TSV and JSON files and reorganizes files into the required directory layout, with explicit support for the \texttt{ecephys} and \texttt{icephys} datatypes defined in this extension.
	\item \textbf{Complete metadata.}
	      Review and complete the generated metadata to verify that probe coordinates, electrode positions, and channel mappings are consistent and to fill in any information not captured in the source files (for example, stereotaxic surgical coordinates).
	\item \textbf{Validate.}
	      Validate the dataset using the BIDS validator, which is schema-driven and will support microephys datasets once the schema changes from this extension are merged into the core BIDS specification.

\end{enumerate}

\section*{Results}

\subsection*{Microephys-BIDS Summary}

The microephys extension introduces two datatypes: \texttt{ecephys} for extracellular recordings (silicon probes, tetrode arrays, Utah arrays) and \texttt{icephys} for intracellular recordings (patch-clamp, sharp electrode).
Both use NWB or NIX as raw data formats.
Data files reside in modality-specific folders (\texttt{ecephys/} or \texttt{icephys/}) with accompanying TSV/JSON metadata following standard BIDS inheritance principles.

\paragraph*{Spatial Metadata:}
Extracellular recordings present a spatial metadata challenge because a multi-channel probe is both a manufactured device with fixed electrode geometry and an implanted object at a specific brain location.
Spike sorting algorithms require the former, i.e., inter-electrode spacing, shank arrangement, and contact impedances, to reconstruct spike waveforms and assign them to recording sites.
Anatomical interpretation requires the latter, i.e., stereotaxic coordinates, brain region labels, and atlas registration, to relate spiking activity to circuit function.
The extension therefore organizes metadata in a three-level hierarchy: 
\begin{enumerate}[noitemsep]
    \item \texttt{probes.tsv} describes devices and their surgical placement, 
    \item \texttt{electrodes.tsv} provides contact positions in probe-relative coordinates (required), and 
    \item \texttt{channels.tsv} links recorded signals to those contacts.
\end{enumerate}
\noindent Anatomical coordinates are optional and distinguished by \texttt{space-<label>} prefixes (for example, \texttt{space-AllenCCFv3\_electrodes.tsv}).
For intracellular recordings, \texttt{samples.tsv} links individual recordings to characterized cells or tissue slices, inheriting animal metadata conventions from Microscopy-BIDS\citep{bourget_microscopy-bids_2022}.

\paragraph*{Animal Subject Metadata:}
Microephys datasets support both human and non-human subjects. Non-human subjects require animal-specific metadata in \texttt{participants.tsv}.
The \texttt{species}, \texttt{strain}, and \texttt{strain\_rrid} columns introduced by Microscopy-BIDS~\citep{bourget_microscopy-bids_2022} are reused here without modification.


\subsection*{Specific Microephys-BIDS Considerations}

\subsection*{Intracellular vs Extracellular Datatypes}
% https://github.com/bids-standard/bids-specification/issues/1800
% Discussion doc: https://docs.google.com/document/d/1GlHr3Go5sR2o_zmxyCU6Sq1W1LLAaV_XZcKpeG_iBFw/edit

Microelectrode electrophysiology (microephys) constitutes a single modality, electrical measurements made with micrometer-scale electrodes, distinguishing it from related BIDS modalities (EEG, MEG, iEEG) that use larger electrodes. Microephys covers two different techniques: extracellular electrophysiology (\texttt{ecephys}), and intracellular electrophysiology  (\texttt{icephys}). In extracellular electrophysiology, electrodes remain in the extracellular space, recording field potentials in the microvolt range from populations of nearby neurons without directly accessing any single cell.
In intracellular electrophysiology, electrodes penetrate or attach to cell membranes, providing direct access to intracellular voltage (tens of millivolts) or current with sub-millisecond temporal resolution of individual synaptic events and intrinsic membrane properties.
This distinction is well-established across electrophysiology data standards: NWB organizes its schema into separate \texttt{icephys} and \texttt{ecephys} modules, openMINDS defines ExtracellularElectrophysiology as a distinct technique in its terminology service, and the Ontology for Biomedical Investigations (OBI) classifies them as separate assay types.
We adopt this distinction at the datatype level, creating \texttt{ecephys} and \texttt{icephys} as two datatypes within the microelectrode electrophysiology modality, analogous to how MRI in BIDS encompasses T1w, T2w, and BOLD.
In the accompanying examples, repeated task trials are distinguished using \texttt{run-<index>}, while the specification supports \texttt{acq-<label>} for different recording configurations within a session.

The different techniques require different metadata.
Spike sorting algorithms run on ecephys data require probe geometry (such as inter-electrode spacing, shank arrangement, contact impedance) to reconstruct waveforms from voltage deflections distributed across contacts and assign action potentials to individual neurons.
A tetrode array's four closely-spaced wires enable sorting by differential geometry; a Neuropixels probe with 384 contacts and known inter-site spacing allows isolation of hundreds of neurons from overlapping electrical signatures.
Intracellular recordings instead require documentation of the cell or tissue: pipette solution composition, seal resistance, recording mode (current-clamp, voltage-clamp), and the relationship between each acquisition and the biological sample (acute slice, organotypic culture, in vivo preparation).
Patch-clamp experiments often involve multiple cells from the same tissue slice or multiple slices from the same animal, requiring explicit tracking from participant to sample to acquisition.

Both datatypes share the core organizational structure of probes, electrodes, and channels, but populate it with technique-specific metadata.
For \texttt{ecephys}, \texttt{probes.tsv} documents surgical placement and device specifications, \texttt{electrodes.tsv} specifies contact layout for spike sorting, and optional ProbeInterface integration links to detailed probe specification.
For \texttt{icephys}, \texttt{samples.tsv} (inherited from Microscopy-BIDS~\citep{bourget_microscopy-bids_2022}) links acquisitions to characterized biological samples with fields such as \texttt{sample\_type}, \texttt{derived\_from}, and \texttt{pathology}.
Channel \texttt{type} indicates the types of data recorded on each channel, and supports values relevant to \texttt{icephys}, such as membrane voltage (\texttt{VM}) and current (\texttt{IM}), as well as values relevant to \texttt{ecephys}, such as local field potential (\texttt{LFP}), high-pass filtered (\texttt{HP}), and multi-unit activity (\texttt{MUA}).

\subsection*{Data Format Requirements}

Following the approach of prior BIDS electrophysiology extensions~\citep{pernet_eeg-bids_2019, holdgraf_ieeg-bids_2019}, our selection of data formats was governed by three requirements:

\begin{enumerate}[noitemsep]
	\item Address the needs of a substantial portion of the electrophysiology community.
	\item Satisfy FAIR principles with open documentation and implementations in widely-used programming languages.
	\item Meet the technical demands of neurophysiological workflows, requiring high numerical precision, efficient storage of large data volumes, efficient access to subsets of large files, and the capacity to store rich metadata alongside raw signals.
\end{enumerate}

We require data to be stored in either NWB~\citep{rubel_neurodata_2022} or NIX~\citep{adrian_file_2014}.
Both are HDF5-based, open-source (BSD-3 licensed), and self-documenting, with the file itself containing the schema needed to interpret its contents.
HDF5 offers practical advantages for multi-gigabyte recordings, supporting hierarchical organization, built-in compression, and lazy loading without reading entire files into memory.

NWB and NIX represent complementary design philosophies.
NWB provides a well-defined schema with extensive validation, ensuring consistency across datasets.
NIX uses a minimal, generic data model that gives users more freedom in how they organize data, and serves as a storage backend for the Neo electrophysiology library~\citep{garcia_neo_2014}.
To ensure interoperability, the BEP32 extension requires that \texttt{.nix} files follow Neo's internal structure conventions, where Neo's object hierarchy and type annotations serve as the domain-specific schema that NIX itself does not prescribe.
Both have been reviewed and endorsed by the INCF as standard practices~\citep{martone_pcall_2019, noauthor_nix_nodate}, and a dedicated archival repository (DANDI, \rrid{RRID:SCR\_017571}) exists for neurophysiology data stored in these formats.

In iEEG-BIDS~\citep{holdgraf_ieeg-bids_2019}, NWB was listed as an unofficially-supported format due to its then-nascent ecosystem.
The maturation of NWB 2.0 and its subsequent INCF endorsement justify its promotion to fully compliant status here.
Unlike iEEG-BIDS, which maintains separate tiers of compliant and unofficially-supported formats, we adopt a simpler two-format approach.
Both NWB and NIX are fully supported with no unofficial tier.

To support the organization of complex data collection and ensure a complete description of experiments, BIDS may require metadata that are also provided in NWB and NIX files to ensure complete provenance of recording sessions within a self-contained container. The presented extension explicitly allows such metadata to appear in both places (i.e., within NWB/NIX files and BIDS metadata fields). This ensures that tools designed for the different standards (NWB, NIX, and BIDS) can function independently. This also simplifies data conversion as tools like nwb2bids can extract metadata directly from the source data to automatically populate corresponding fields in BIDS. Also, SpikeInterface can already extract probe geometry from NWB files, and future integration of SpikeInterface with BIDS sidecars will provide similar functionality for ecephys data in BIDS.

\subsection*{Probes, Electrodes, and Channels}

Although the NIX and NWB files can contain all information about the channels, electrode placement and probe design, in the BIDS representation these are explicitly made available in human- and machine-readable format in tabular TSV and in JSON files.
Microephys uses three tables to capture the metadata of an experiment: \texttt{probes.tsv}, \texttt{electrodes.tsv}, and \texttt{channels.tsv} (and the corresponding \texttt{.json} files).
The \texttt{probes.tsv} file lists each probe with its type and surgical placement.
The \texttt{electrodes.tsv} file enumerates contact sites, linking each to its probe and specifying probe-relative coordinates for spike sorting.
The \texttt{channels.tsv} file documents recorded signals: sampling frequency, units, source electrode, and reference.
Channels without electrodes (sync, stimulus) use \texttt{n/a}; ground, typically hardware like a skull screw or ear clip, is defined globally in the sidecar JSON and can be overridden per-channel if it differs.
A \texttt{stream\_id} column links channels to data objects within the NWB or NIX file.

The iEEG-BIDS and EEG-BIDS extensions distinguish electrodes from channels: an electrode is a contact point with tissue, a channel is the amplifier and digitizer that produces a stored time series.
These are not equivalent and are not always one-to-one.
A differential recording derives one channel from two electrodes, a Neuropixels electrode may yield separate highpass and LFP channels at different sampling rates, and a sync signal is a channel with no electrode at all.
The prior extensions noted that electrodes can be ``organized into arrays, grids, or probes,'' but their metadata captured only electrodes and channels, not this grouping.

We add an explicit probe level.
A probe is the physical device that carries one or more electrodes, such as a Neuropixels probe, tetrode bundle, patch pipette probe.
The Probe table stores metadata about a single or multiple probes, such as manufacturer, surgical coordinates, and insertion angle.
It provides a mechanism to link to formal probe model specifications.

\subsection*{ProbeInterface Integration}

In extracellular electrophysiology, the spatial layout of the electrodes (i.e.\ contacts) on the probe is important for data processing (such as via spike sorting), and in particular on high-density probes~\citep{buccino_spikeinterface_2020}.
The positions of the electrodes on a template brain can be useful as well, but often the relative positions of the electrodes are substantially warped in the registration process, so microephys needs to also store relative positions of the electrodes on the probe.
The ProbeInterface library (\rrid{RRID:SCR\_021150}, part of the SpikeInterface project)~\citep{garcia_probeinterface_2022} provides a JSON-based schema for specifying the layout of electrodes on a probe, as well as for specifying the shapes of the electrodes and the geometry of the probe itself, which may be used for further visualization and analysis (Figure~\ref{fig:probeinterface}).

The \texttt{ecephys} datatype provides three possible mechanisms to provide the layout of the electrodes:
\begin{enumerate}[noitemsep]
	\item \textbf{\texttt{electrodes.tsv} (required):}
	      An \texttt{electrodes.tsv} file without the use of a \texttt{space} entity is required, and this file uses \texttt{x}, \texttt{y}, and sometimes \texttt{z} to specify the positions of each electrode relative to the probe tip.
	      This data is present for every electrophysiology data stream, and can be used by downstream tools.
	      Electrodes in an alternative space may be defined in \texttt{electrodes.tsv} files with the \texttt{space} entity defined.

	\item \textbf{Link to probeinterface\_library:}
	      ProbeInterface maintains a library of popular probes.
	      % TODO: fill in current count (currently xxx)
	      If the \texttt{ecephys} is recorded from a probe that is defined in this library, the user is encouraged to link to the formal ProbeInterface definition of the probe, which contains additional information about the probe, including the electrode shape, and probe shape.
	      This link uses the \texttt{Levels} mechanism with \texttt{TermURL} to link from values of the \texttt{probe\_model} column of the \texttt{probes.tsv} table to the URL of the probe definition on GitHub.

	\item \textbf{Define custom ProbeInterface files:}
	      If probeinterface\_library does not have the probe used, users are encouraged to create a probe definition JSON file.
	      This file can be placed in \texttt{/probes}, and the \texttt{probe\_model} column of the \texttt{probes.tsv} table can use \texttt{Levels} with a BIDS URI to direct to the file.
	      The user may also consider submitting this probe configuration file to the probeinterface\_library for use in future datasets.
\end{enumerate}

\subsection*{Brain Coordinate Systems}

Like other datatypes, including iEEG, MEG, and EEG, microelectrode electrophysiology must provide a way to report the anatomical localization of electrodes. In practice, there are two main ways to report brain location for microephys. Stereotaxic coordinates define precise locations within the brain relative to an external, fixed skull landmark (e.g., Bregma) or an apparatus, used for targeting, and are commonly used to represent the surgical placement of the probe. Anatomical coordinates instead describe positions based on internal, structural landmarks (like the Anterior Commissure) within brain atlases and may not always be known. In contrast to iEEG, MEG, and EEG, microephys experiments:
\begin{enumerate}[noitemsep]
	\item Often use stereotaxic coordinates to represent the surgical placement of the probe.
	\item Anatomical localization is not always known.
	\item Processing of the data requires understanding of real (unwarped) electrode spacing.
	\item Are often performed on various non-human species.
\end{enumerate}

\subsubsection*{Stereotaxic coordinates}

When inserting microelectrode probes in vivo, experimenters often use stereotaxic instruments to target a desired brain region.
The position and angle of the inserted probe are the stereotaxic coordinates, and describe the surgical positioning procedure.
We reviewed the field thoroughly and found that the systems for notating stereotaxic coordinates were often varied, inconsistent, and poorly documented.
The BrainSTEM project~\citep{petersen_brainstem} provides clear and unambiguous definitions of stereotaxic coordinates along with detailed illustrations in its online documentation.
We adopt and formalize this convention here, and encourage others to do the same.

In microephys, the stereotaxic coordinates are stored as columns of the \texttt{probes.tsv} table.
This includes 3 axes of direction: anterior/posterior (AP), medial/lateral (ML), and dorsal/ventral (DV); and 3 rotational axes: AP Angle, ML Angle, and rotation (i.e.\ twist).

In non-human neuroanatomy, the direction towards the top of the brain is usually referred to as dorsal (D) and the bottom ventral (V).
In human neuroanatomy, these directions are usually called superior and inferior, respectively.
We note that we are using the animal convention (D/V), here, as we expect the majority of stereotaxic microelectrode electrophysiology experiments to be done in non-humans.

Stereotaxic instruments are not always used.
For example, placement of Utah array probes in non-human primates and in humans is often done without a stereotaxic instrument.
They are often simply manually placed in the desired brain area based on gyral patterns observed on the surface of the brain.
In these cases, there are no stereotaxic coordinates to report.

Stereotaxic coordinates are valuable information because they capture the intended surgical probe placement which is often the best estimate the experimenter has until histological probe localization can be conducted post mortem, however, a recent analysis has revealed that stereotaxic reproducibility can be quite poor, even in ideal circumstances, and substantial, meaningful variance exists between surgeries~\citep{international_brain_laboratory_reproducibility_2025}.
Stereotaxic coordinates also fail to capture inter-individual variability in brain structure.
Therefore, while this information is valuable, it is incomplete.

\subsubsection*{Anatomical localization of electrodes}

The precise anatomical positioning of probes and electrodes can be determined using neuroimaging or histology.
The most common method is to take histological slices of the brain where the probe was inserted and visually inspect them to determine where the electrodes were.

The gold standard for anatomical localization in microephys experiments is to register the probe to a 3-dimensional coordinate system.
This technique registers electrodes implanted in different subjects onto a common brain, analogous to registration procedures that are commonplace in MRI.
Most microelectrode electrophysiology data is from non-human animal subjects, so we must provide a new set of coordinate systems for different species.
The most popular coordinate system for mice is the Allen Institute Common Coordinate Framework (CCF) v3~\citep{wang_allen_2020}.
Other atlases have been developed for mice and a variety of other species, though complete coverage of species will require new coordinate systems and atlases to be added as they are developed.

Expressing localization of electrodes on an individual, group, or common coordinate system requires an additional \texttt{electrodes.tsv} file with a \texttt{space} entity, and a corresponding \texttt{MicroephysCoordinateSystem.json} file.
Here we deviate from the iEEG datatype---the main \texttt{electrodes.tsv} file must store the real physical coordinates of electrodes in relative probe-space, as this is important information for downstream processing.

\section*{Discussion}

While NWB and NIX standardize electrophysiology data within individual recording sessions, this extension provides consistent organization across subjects and sessions, making essential metadata available in human-readable, machine-parseable form through TSV and JSON sidecars.
By separating extracellular and intracellular recordings into distinct datatypes, each with technique-specific metadata (probe geometry and electrode coordinates for spike sorting, sample provenance and recording mode for patch clamp), the extension ensures that analysis tools can discover and interpret microephys datasets without parsing the underlying data files.

The specification defines the organization of raw data but does not yet address the most common forms of derived data, particularly spike sorting results and associated quality metrics.
However, the standardized probe geometry enabled by ProbeInterface integration, when consumed by tools such as SpikeInterface, provides a pathway toward standardized spike sorting outputs that complement this specification.

The brain coordinate systems defined in this extension cover the most common animal models (mouse, rat, macaque, marmoset), but electrophysiology is performed in many additional species for which standardized digital brain atlases either do not exist or have not yet reached the level of community adoption required for inclusion.
The specification accommodates unlisted atlases through the \texttt{Other} coordinate system keyword with a free-text description; commonly-used atlases can be promoted to named entries in future specification updates.

NWB and NIX are not the native output format of most commercial acquisition systems.
Of the major platforms (Blackrock, Plexon, Intan, Neuralynx, SpikeGLX, TDT), only Open Ephys supports NWB recording natively.
Researchers may also prefer native formats during acquisition and analysis for performance reasons.
The conversion step, while a barrier to adoption, reflects the same transition neuroimaging underwent from proprietary scanner formats to NIfTI: without a common data format, reusable tools cannot be built across laboratories.
% TODO: quantify format counts if possible (Neo lists 36 RawIO modules as of 2026-03; NeuroConv docs claim ~50)
Conversion tools such as Neo and NeuroConv support reading from dozens of acquisition formats and lower this barrier in practice.

The major neurophysiology repositories already operate on principles that microephys-BIDS complements.
DANDI requires NWB and supports BIDS validation for uploaded datasets.
EBRAINS (\rrid{RRID:SCR\_019260}) recommends BIDS for human neuroimaging data and provides general data organisation guidelines (modality separation, subject-based hierarchy, machine-readable metadata) that align with BIDS conventions; its recommendation has not yet extended to electrophysiology because no animal microelectrode extension existed until now.
Repositories such as GIN (\rrid{RRID:SCR\_015864}) store data with full filesystem fidelity, making the benefits of a harmonized directory structure immediately apparent to both humans and machines.

Adoption of the presented extension would enable BIDS-Apps that exploit the standardized structure and metadata for automated processing workflows, and provide laboratories with a consistent organisational framework.
Domain-specific standards address the unique requirements of individual modalities, but modern neurophysiology experiments increasingly combine electrophysiology with neuroimaging, optical physiology, and behavioral recordings in the same subjects and sessions.
BIDS complements these standards by providing a modality-spanning organisational layer, and with the addition of microelectrode electrophysiology, now covers the full range of modalities common in systems neuroscience laboratories.

Two example datasets are provided in the BIDS examples repository (\url{https://github.com/bids-standard/bids-examples}).
The first is a conversion of the Reach-to-Grasp multi-electrode array dataset~\citep{brochier_massively_2018}, originally published on the G-Node repository (DOI: 10.12751/g-node.f83565).
It contains extracellular recordings from two macaque subjects using 96-channel Utah arrays, with both local field potential (1~kHz) and high-pass (30~kHz) data streams, behavioral channels, and task events.
The second is a synthetic dataset demonstrating both \texttt{ecephys} and \texttt{icephys} datatypes, including registration to the Allen CCF v3 coordinate system, multiple simultaneous probes, and both ProbeInterface library references and custom probe definition files.

\section*{Code Availability}

Using NWB and NIX as the underlying data format allows microephys BIDS data to utilize the sophisticated software ecosystems associated with these data standards.

The NWB software ecosystem contains a suite of tools to convert data from the various common electrophysiology data formats to NWB.
PyNWB (\rrid{RRID:SCR\_017452}) and MatNWB (\rrid{RRID:SCR\_021156}) provide flexible APIs for writing NWB files~\citep{rubel_neurodata_2022}.
NeuroConv automates conversion from 60+ formats, guiding the user in best practices~\citep{mayorquin_neuroconv_2025}.
NWB Inspector (\rrid{RRID:SCR\_025465}) provides automated checks to validate NWB files against best practices, and NWB GUIDE provides a graphical interface that provides the features of NeuroConv and NWB Inspector to non-programmers.

30+ popular analysis tools support NWB as an input and/or output format, many of which are relevant to microephys data.
SpikeInterface provides tools for creating spike sorting pipelines~\citep{buccino_spikeinterface_2020}.
Neurosift (\rrid{RRID:SCR\_025466}) provides interactive dashboards for visualizing NWB data~\citep{magland_neurosift_2024}, and Pynapple (\rrid{RRID:SCR\_023312})~\citep{viejo_pynapple_2023} provides tools for common neurophysiology analyses.

NIX can be written from NEO~\citep{garcia_neo_2014}, which supports reading diverse electrophysiology data formats.
NIX also supports data analysis libraries, such as Elephant (\rrid{RRID:SCR\_003833})~\citep{denker_collaborative_2018}.
