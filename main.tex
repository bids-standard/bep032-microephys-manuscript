\begin{abstract}
	The Brain Imaging Data Structure (BIDS) has facilitated data sharing and tool development in human neuroimaging.
	We present an extension for microelectrode electrophysiology recordings in animals, specifically addressing the unique requirements of intracellular and extracellular recordings.
	This extension introduces two new data types: \texttt{icephys} for intracellular and \texttt{ecephys} for extracellular recordings, supporting diverse recording modalities from patch-clamp to high-density silicon probes.
	Building on existing BIDS principles and the prior electrophysiology extensions for EEG and iEEG, we specify metadata for probes, electrodes, and channels, paying particular attention to metadata required for spike sorting analysis.
	The extension adopts NWB (Neurodata Without Borders) and NIX (Neuroscience Information Exchange) as data formats, ensuring comprehensive metadata capture while maintaining compatibility with existing analysis ecosystems.
	We provide example datasets covering common use cases and discuss integration with established tools.
	This standardization enables reproducible analysis pipelines, facilitates data sharing through repositories like DANDI, G-Node and EBRAINS, and bridges different scales from cellular to systems neuroscience.
\end{abstract}

\section*{Introduction}

Microelectrode electrophysiology encompasses techniques for recording electrical activity on spatial scales smaller than one millimetre, such as the membrane potential of individual neurons and local field potentials, providing crucial insights into neural computation at high spatial and temporal resolution, often with the ability to track individual neuronal action potentials.
Recent technological advances, including high-density silicon probes such as Neuropixels~\citep{jun_fully_2017}, have dramatically increased data acquisition rates, with single probes now capable of recording from hundreds of neurons simultaneously.
Intracellular techniques such as patch clamp recordings~\citep{sakmann_patch_1984} provide direct access to cellular membrane dynamics, enabling detailed characterization of synaptic transmission and intrinsic neuronal properties.
These methods are primarily used in animal research and generate data of increasing complexity that demands standardized organization.

A key challenge in the microelectrode electrophysiology community has been the proliferation of proprietary data formats.
Different acquisition systems, such as Blackrock, Plexon, TDT, Open Ephys, and SpikeGLX, each produce data in their own formats, creating barriers to data sharing and tool interoperability.
This fragmentation complicates data sharing and long-term archival: custom readers must be maintained for each format (see for example the Neo (\rrid{RRID:SCR\_000634}) library~\citep{garcia_neo_2014}.)
Undocumented specifications, unmaintained programmatic interfaces, and language lock-in risk rendering data inaccessible.

Two major community efforts have emerged to address this challenge at the file format level for single recording sessions.
Neurodata Without Borders (NWB, \rrid{RRID:SCR\_015242})~\citep{rubel_neurodata_2022, teeters_neurodata_2015} is an open data standard for neurophysiology developed as part of the National Institutes of Health BRAIN Initiative by a community of researchers over the last 10 years.
NWB uses HDF5 or Zarr as its backend and provides a comprehensive schema for storing electrophysiology data alongside stimulus information, behavioral data, and metadata.
The Neuroscience Information Exchange Format (NIX, \rrid{RRID:SCR\_016196})~\citep{adrian_file_2014} is a data model for annotated scientific datasets that provides an HDF5 file backend for the Neo library.
Both standards are widely adopted in the neuroscience community and supported by major analysis toolkits such as SpikeInterface (\rrid{RRID:SCR\_021150})~\citep{buccino_spikeinterface_2020}, and both have been endorsed by the International Neuroinformatics Coordinating Facility (INCF) as community standards for electrophysiology data.
Both NWB and NIX have been successful in standardizing data representation at the single-session level, enabling tool developers to build analysis pipelines that work across laboratories~\citep{denker_collaborative_2018, viejo_pynapple_2023, magland_neurosift_2024}.
However, neither standard provides organizational guidance at the project level.
That is, how to organize data across multiple sessions, subjects, and experimental conditions within a study.
Researchers must still make ad-hoc decisions about directory structures, naming conventions, and where to place study-level metadata such as participant information or experimental protocols.

The Brain Imaging Data Structure (BIDS, \rrid{RRID:SCR\_016124})~\citep{gorgolewski_brain_2016} is a community-driven standard for organizing and sharing neuroscience data.
In a BIDS dataset, all data from each subject are organized hierarchically in subject-specific directories, with optional session subdirectories containing modality-specific folders for the data files themselves.
Files follow a structured naming scheme that encodes relevant metadata such as subject, session, and task.
JSON sidecar files provide detailed metadata for each data file, while TSV files capture tabular information such as participant demographics and channel properties.
A \texttt{dataset\_description.json} file describes the dataset as a whole, and JSON metadata at higher directory levels are inherited by lower levels unless overridden.
Originally developed for MRI, BIDS has been extended through community-driven proposals to additional modalities including MEG~\citep{niso_meg-bids_2018}, EEG~\citep{pernet_eeg-bids_2019}, iEEG~\citep{holdgraf_ieeg-bids_2019}, PET, and microscopy.
With over a thousand public datasets on OpenNeuro and endorsement by the INCF, BIDS has become a widely adopted framework for FAIR neuroscience data.
The existing electrophysiology extensions established patterns for organizing time-series neural data that we build upon here.
This extension bridges the gap between session-level data standardization (NWB/NIX) and study-level organization (BIDS), combining the strengths of both approaches.
BIDS provides the organizational framework for datasets containing multiple subjects and sessions, consistent with existing modalities, while NWB and NIX handle the complex internal structure of electrophysiology recordings within each session.
This complementary relationship mirrors how BIDS for neuroimaging organizes NIfTI files.
BIDS does not replace the data format but provides the organizational context around it.

While we tried wherever possible to maintain consistency with existing datatypes, we also designed this enhancement proposal to address the unique challenges of microelectrode electrophysiology:
\begin{enumerate}
	\item Electrode scales spanning orders of magnitude (sub-micron tips to millimeter arrays).
	\item Diverse target species, which have different brain atlases, often including entirely different brain areas.
	\item Large data volumes, which can be over 100~GB for an individual session.
	\item Diverse probe geometries requiring specialized coordinate systems.
	\item Spike sorting as an essential preprocessing step requires specific metadata.
\end{enumerate}

Here we present BIDS Extension Proposal 032 (BEP032), extending BIDS to microelectrode electrophysiology with a focus on animal models.
This extension:
\begin{itemize}
	\item introduces two datatypes (\texttt{ecephys} for extracellular and \texttt{icephys} for intracellular recordings),
	\item defines metadata across three levels: probes, electrodes, and channels,
	\item standardizes probe and electrode metadata including stereotaxic surgical coordinates,
	\item integrates with the ProbeInterface library~\citep{garcia_probeinterface_2022} for probe geometry definitions, and
	\item defines coordinate systems for multiple species including rodent and primate brain atlases.
\end{itemize}

\subsection*{Standardization landscape}

The MINI checklist~\citep{gibson_minimum_2009}, developed by the CARMEN consortium, was among the first community efforts to define what metadata should accompany electrophysiology recordings, covering subject, electrode, recording location, and stimulus information, but explicitly left data format and organization out of scope.
odML (\rrid{RRID:SCR\_001376})~\citep{grewe_bottom-up_2011} addressed how to structure such metadata through flexible key-value annotation and was adopted in practice for multi-electrode array datasets~\citep{zehl_handling_2016}.
NWB~\citep{teeters_neurodata_2015} and NIX~\citep{adrian_file_2014} then standardized the data format itself, embedding metadata within self-describing files, while Neo~\citep{garcia_neo_2014} provided the programmatic IO layer that bridges proprietary acquisition formats to these open standards.
At the project level, large-scale collaborations developed their own organizational conventions: the Allen Brain Observatory~\citep{de_vries_sharing_2023} adopted internal data management practices, while the International Brain Laboratory designed the Open Neurophysiology Environment (ONE) to coordinate Neuropixels recordings across dozens of laboratories.
BEP032 occupies this project-level layer, providing a community standard that builds on the mature ecosystems from these earlier initiatives and that could serve as a common baseline for such collaborative efforts.

\section*{Results}

\subsection*{Microephys-BIDS Summary}

The microephys extension introduces two datatypes: \texttt{ecephys} for extracellular recordings (silicon probes, tetrode arrays, Utah arrays) and \texttt{icephys} for intracellular recordings (patch-clamp, sharp electrode).
Both use NWB or NIX as raw data formats.
Data files reside in modality-specific folders (\texttt{ecephys/} or \texttt{icephys/}) with accompanying TSV/JSON metadata following standard BIDS inheritance principles.

Extracellular recordings present a spatial metadata challenge: a multi-channel probe is both a manufactured device with fixed electrode geometry and an implanted object at a specific brain location.
Spike sorting algorithms require the former—inter-electrode spacing, shank arrangement, contact impedances—to reconstruct spike waveforms and assign them to recording sites.
Anatomical interpretation requires the latter—stereotaxic coordinates, brain region labels, atlas registration—to relate spiking activity to circuit function.
The extension therefore organizes metadata in a three-level hierarchy: \texttt{probes.tsv} describes devices and their surgical placement, \texttt{electrodes.tsv} provides contact positions in probe-relative coordinates (required), and \texttt{channels.tsv} links recorded signals to those contacts.
Anatomical coordinates are optional and distinguished by \texttt{space-<label>} prefixes (for example, \texttt{space-AllenCCFv3\_electrodes.tsv}).
For intracellular recordings, \texttt{samples.tsv} links individual recordings to characterized cells or tissue slices, inheriting animal metadata conventions from Microscopy-BIDS\citep{bourget_microscopy-bids_2022}.

\subsubsection*{Animal subject metadata}

Microephys datasets predominantly involve non-human subjects, requiring animal-specific metadata in \texttt{participants.tsv}.
The \texttt{species}, \texttt{strain}, and \texttt{strain\_rrid} columns introduced by Microscopy-BIDS~\citep{bourget_microscopy-bids_2022} are reused here without modification.
An \texttt{age\_category} column using UBERON life-cycle stage terms is under development in a separate specification effort (PR~\#2340).

\subsection*{Specific Microephys-BIDS Considerations}

\subsection*{Intracellular vs Extracellular Datatypes}
% https://github.com/bids-standard/bids-specification/issues/1800
% Discussion doc: https://docs.google.com/document/d/1GlHr3Go5sR2o_zmxyCU6Sq1W1LLAaV_XZcKpeG_iBFw/edit

Microelectrode electrophysiology constitutes a single modality—electrical measurements made with micrometer-scale electrodes, distinguishing it from related BIDS modalities (EEG, MEG, iEEG) that use larger electrodes—but encompasses two fundamentally different acquisition techniques.
Extracellular electrodes remain in the extracellular space, recording field potentials in the microvolt range from populations of nearby neurons without directly accessing any single cell.
Intracellular electrodes penetrate or attach to cell membranes, providing direct access to intracellular voltage (tens of millivolts) or current with sub-millisecond temporal resolution of individual synaptic events and intrinsic membrane properties.
This distinction is well-established across the field: NWB organizes its schema into separate \texttt{icephys} and \texttt{ecephys} modules, openMINDS defines ExtracellularElectrophysiology as a distinct technique in its terminology service, and the Ontology for Biomedical Investigations (OBI) classifies them as separate assay types.
We adopt this distinction at the datatype level, creating \texttt{ecephys} and \texttt{icephys} as two datatypes within the microelectrode electrophysiology modality, analogous to how MRI encompasses T1w, T2w, and BOLD.
In the accompanying examples, repeated task trials are distinguished using \texttt{run-<index>}, while the specification supports \texttt{acq-<label>} for different recording configurations within a session.

The different techniques demand different metadata.
Spike sorting algorithms require probe geometry—inter-electrode spacing, shank arrangement, contact impedance—to reconstruct waveforms from voltage deflections distributed across contacts and assign action potentials to individual neurons.
A tetrode array's four closely-spaced wires enable sorting by differential geometry; a Neuropixels probe with 384 contacts and known inter-site spacing allows isolation of hundreds of neurons from overlapping electrical signatures.
Intracellular recordings instead require documentation of the cell or tissue: pipette solution composition, seal resistance, recording mode (current-clamp, voltage-clamp), and the relationship between each acquisition and the biological sample (acute slice, organotypic culture, in vivo preparation).
Patch-clamp experiments often involve multiple cells from the same tissue slice or multiple slices from the same animal, requiring explicit tracking from participant to sample to acquisition.

Both datatypes share the core organizational structure--probes, electrodes, channels--but populate it with technique-specific metadata.
For \texttt{ecephys}, \texttt{probes.tsv} documents surgical placement and device specifications, \texttt{electrodes.tsv} specifies contact layout for spike sorting, and optional ProbeInterface integration links to detailed probe specification.
For \texttt{icephys}, \texttt{samples.tsv} (inherited from Microscopy-BIDS~\citep{bourget_microscopy-bids_2022}) links acquisitions to characterized biological samples with fields such as \texttt{sample\_type}, \texttt{derived\_from}, and \texttt{pathology}.
Channel \texttt{type} values distinguish membrane voltage and current (\texttt{VM}, \texttt{IM}) from extracellular signals (\texttt{LFP}, \texttt{HP}, \texttt{MUA}).

\subsection*{Data Format Requirements}

Following the approach of prior BIDS electrophysiology extensions~\citep{pernet_eeg-bids_2019, holdgraf_ieeg-bids_2019}, our selection of data formats was governed by three requirements:

\begin{enumerate}
	\item address the needs of a substantial portion of the electrophysiology community,
	\item satisfy FAIR principles with open documentation and implementations in widely-used programming languages, and
	\item meet the technical demands of neurophysiological workflows: high numerical precision, efficient storage of large data volumes, efficient access to subsets of large files, and the capacity to store rich metadata alongside raw signals.
\end{enumerate}

We require data to be stored in NWB~\citep{rubel_neurodata_2022} or NIX~\citep{adrian_file_2014}.
Both are HDF5-based, open-source (BSD-3 licensed), and self-documenting: the file itself contains the schema needed to interpret its contents.
HDF5 offers practical advantages for multi-gigabyte recordings: hierarchical organization, built-in compression, and lazy loading without reading entire files into memory.

These two formats represent complementary design philosophies.
NWB provides a well-defined schema with extensive validation, ensuring consistency across datasets.
NIX uses a minimal, generic data model that gives users more freedom in how they organize data, and serves as a storage backend for the Neo electrophysiology library~\citep{garcia_neo_2014}.
To ensure interoperability, this extension requires that \texttt{.nix} files follow Neo's internal structure conventions, where Neo's object hierarchy and type annotations serve as the domain-specific schema that NIX itself does not prescribe.
Both have been reviewed and endorsed by the INCF as standard practices~\citep{martone_pcall_2019, noauthor_nix_nodate}, and a dedicated archival repository (DANDI, \rrid{RRID:SCR\_017571}) exists for neurophysiology data stored in these formats.

In iEEG-BIDS~\citep{holdgraf_ieeg-bids_2019}, NWB was listed as an unofficially-supported format due to its then-nascent ecosystem.
The maturation of NWB 2.0 and its subsequent INCF endorsement justify its promotion to fully compliant status here.
Unlike iEEG-BIDS, which maintains separate tiers of compliant and unofficially-supported formats, we adopt a simpler two-format approach.
Both NWB and NIX are fully supported with no unofficial tier.

BIDS provides dataset-level organization: directory structure, naming conventions, and human-readable TSV/JSON files that allow inspection without specialized tools.
The NWB or NIX file ensures complete provenance within a self-contained container.
Metadata may appear in both places; analysis tools such as SpikeInterface can already extract probe geometry from NWB files, and future integration with BIDS sidecars will provide an alternative pathway.

\subsection*{Probes, Electrodes, and Channels}

Although the NIX and NWB files can contain all information about the channels, electrode placement and probe design, in the BIDS representation these are explicitly made available in human- and machine-readable format in tabular TSV and in JSON files.
Microephys uses three tables to capture the metadata of an experiment: \texttt{probes.tsv}, \texttt{electrodes.tsv}, and \texttt{channels.tsv} (and the corresponding \texttt{.json} files).
The \texttt{probes.tsv} file lists each probe with its type and surgical placement.
The \texttt{electrodes.tsv} file enumerates contact sites, linking each to its probe and specifying probe-relative coordinates for spike sorting.
The \texttt{channels.tsv} file documents recorded signals: sampling frequency, units, source electrode, and reference.
Channels without electrodes (sync, stimulus) use \texttt{n/a}; ground, typically hardware like a skull screw or ear clip, is defined globally in the sidecar JSON and can be overridden per-channel if it differs.
A \texttt{stream\_id} column links channels to data objects within the NWB or NIX file.

The iEEG-BIDS and EEG-BIDS extensions distinguish electrodes from channels: an electrode is a contact point with tissue, a channel is the amplifier and digitizer that produces a stored time series.
These are not equivalent and are not always one-to-one.
A differential recording derives one channel from two electrodes, a Neuropixels electrode may yield separate highpass and LFP channels at different sampling rates, and a sync signal is a channel with no electrode at all.
The prior extensions noted that electrodes can be ``organized into arrays, grids, or probes,'' but their metadata captured only electrodes and channels, not this grouping.

We add an explicit probe level.
A probe is the physical device---Neuropixels, tetrode bundle, patch pipette---carrying one or more electrodes.
The Probe table stores metadata about a single or multiple probes, such as manufacturer, surgical coordinates, and insertion angle.
It provides a mechanism to link to formal probe model specifications.

\subsection*{ProbeInterface Integration}

In extracellular electrophysiology, the spatial layout of the electrodes (i.e.\ contacts) on the probe is important for processing such as spike sorting, particularly on high-density probes~\citep{buccino_spikeinterface_2020}.
The positions of the electrodes on a template brain can be useful as well, but often the relative positions of the electrodes are substantially warped in the registration process, so microephys needs to also store relative positions of the electrodes on the probe.
The ProbeInterface library (\rrid{RRID:SCR\_021150}, part of the SpikeInterface project)~\citep{garcia_probeinterface_2022} provides a JSON-based schema for specifying electrode layout on the probe, as well as specifying the shapes of the electrodes and the geometry of the probe itself, which may be used for further visualization and analysis (Figure~\ref{fig:probeinterface}).

The \texttt{ecephys} datatype provides three possible mechanisms to provide the layout of the electrodes:
\begin{enumerate}
	\item \textbf{\texttt{electrodes.tsv} (required):}
	      An \texttt{electrodes.tsv} file without the use of a \texttt{space} entity is required, and this file uses \texttt{x}, \texttt{y}, and sometimes \texttt{z} to specify the positions of each electrode relative to the probe tip.
	      This data is present for every electrophysiology data stream, and can be used by downstream tools.
	      Electrodes in an alternative space may be defined in \texttt{electrodes.tsv} files with the \texttt{space} entity defined.

	\item \textbf{Link to probeinterface\_library:}
	      ProbeInterface maintains a library of popular probes.
	      % TODO: fill in current count (currently xxx)
	      If the \texttt{ecephys} is recorded from a probe that is defined in this library, the user is encouraged to link to the formal ProbeInterface definition of the probe, which contains additional information about the probe, including the electrode shape, and probe shape.
	      This link uses the \texttt{Levels} mechanism with \texttt{TermURL} to link from values of the \texttt{probe\_model} column of the \texttt{probes.tsv} table to the URL of the probe definition on GitHub.

	\item \textbf{Define custom ProbeInterface files:}
	      If probeinterface\_library does not have the probe used, users are encouraged to create a probe definition JSON file.
	      This file can be placed in \texttt{/probes}, and the \texttt{probe\_model} column of the \texttt{probes.tsv} table can use \texttt{Levels} with a BIDS URI to direct to the file.
	      The user may also consider submitting this probe configuration file to the probeinterface\_library for use in future datasets.
\end{enumerate}

\subsection*{Coordinate Systems}

Like other datatypes, including iEEG, MEG, and EEG, microelectrode electrophysiology must provide a way to report the anatomical localization of electrodes.
However, this standard has four key differences in microephys experiments:
\begin{enumerate}
	\item They often use stereotaxic coordinates to represent the surgical placement of the probe.
	\item Anatomical localization is not always known.
	\item Processing of the data requires understanding of real (unwarped) electrode spacing.
	\item The experiments are often done on various non-human species.
\end{enumerate}

\subsubsection*{Stereotaxic coordinates}

When inserting microelectrode probes in vivo, experimenters often use stereotaxic instruments to target a desired brain region.
The position and angle of the inserted probe are the stereotaxic coordinates, and describe the surgical positioning procedure.
We reviewed the field thoroughly, and found that the systems for notating stereotaxic coordinates were often varied, inconsistent, and poorly documented.
The BrainSTEM project~\citep{petersen_brainstem} provides clear and unambiguous definitions of stereotaxic coordinates along with detailed illustrations in its online documentation.
We adopt and formalize this convention here, and encourage others to do the same.

In microephys, the stereotaxic coordinates are stored as columns of the \texttt{probes.tsv} table.
This includes 3 axes of direction: anterior/posterior (AP), medial/lateral (ML), and dorsal/ventral (DV); and 3 rotational axes: AP Angle, ML Angle, and rotation (i.e.\ twist).

In non-human neuroanatomy, the direction towards the top of the brain is usually referred to as dorsal (D) and the bottom ventral (V).
In human neuroanatomy, these directions are usually called superior and inferior, respectively.
We note that we are using the animal convention (D/V), here, as we expect the majority of stereotaxic microelectrode electrophysiology experiments to be done in non-humans.

Stereotaxic instruments are not always used.
For example, placement of Utah array probes in non-human primates and in humans is often done without a stereotaxic instrument.
They are often simply manually placed in the desired brain area based on gyral patterns observed on the surface of the brain.
In these cases, there are no stereotaxic coordinates to report.

Stereotaxic coordinates are valuable information, as it reveals the intent of the surgical probe placement, and it is often the best estimate the experimenter has until histological probe localization can be conducted post mortem, however, a recent analysis has revealed that stereotaxic reproducibility can be quite poor, even in ideal circumstances, and substantial, meaningful variance exists between surgeries~\citep{international_brain_laboratory_reproducibility_2025}.
Stereotaxic coordinates also fail to capture inter-individual variability in brain structure.
Therefore, while this information is valuable, it is incomplete.

\subsection*{Anatomical localization of electrodes}

The precise anatomical positioning of probes and electrodes can be determined using neuroimaging or histology.
The most common method is to take histological slices of the brain where the probe was inserted and visually inspect them to determine where the electrodes were.

The gold standard for anatomical localization in microephys experiments is to register the probe to a 3-dimensional coordinate system.
This technique registers electrodes implanted in different subjects onto a common brain, analogous to registration procedures that are commonplace in MRI.
Most microelectrode electrophysiology data is from non-human animal subjects, so we must provide a new set of coordinate systems for different species.
The most popular coordinate system for mice is the Allen Institute Common Coordinate Framework (CCF) v3~\citep{wang_allen_2020}.
Other atlases have been developed for mice and a variety of other species, though complete coverage of species will require new coordinate systems and atlases to be added as they are developed.

Expressing localization of electrodes on an individual, group, or common coordinate system requires an additional \texttt{electrodes.tsv} file with a \texttt{space} entity, and a corresponding \texttt{MicroephysCoordinateSystem.json} file.
Here we deviate from the iEEG datatype---the main \texttt{electrodes.tsv} file must store the real physical coordinates of electrodes in relative probe-space, as this is important information for downstream processing.

\section*{Discussion}

NWB and NIX standardize electrophysiology data within individual recording sessions; this extension provides consistent organization across subjects and sessions, making essential metadata available in human-readable, machine-parseable form through TSV and JSON sidecars.
By separating extracellular and intracellular recordings into distinct datatypes, each with technique-specific metadata (probe geometry and electrode coordinates for spike sorting, sample provenance and recording mode for patch clamp), the extension ensures that analysis tools can discover and interpret microephys datasets without parsing the underlying data files.

The specification defines the organization of raw data but does not yet address the most common forms of derived data, particularly spike sorting results and associated quality metrics.
However, the standardized probe geometry enabled by ProbeInterface integration, when consumed by tools such as SpikeInterface, provides a pathway toward standardized spike sorting outputs that complement this specification.

The coordinate systems defined in this extension cover the most common animal models (mouse, rat, macaque, marmoset), but electrophysiology is performed in many additional species for which standardized digital brain atlases either do not exist or have not yet reached the level of community adoption required for inclusion.
The specification accommodates unlisted atlases through the \texttt{Other} coordinate system keyword with a free-text description; commonly-used atlases can be promoted to named entries in future specification updates.

NWB and NIX are not the native output format of most commercial acquisition systems.
Of the major platforms (Blackrock, Plexon, Intan, Neuralynx, SpikeGLX, TDT), only Open Ephys supports NWB recording natively.
Researchers may also prefer native formats during acquisition and analysis for performance reasons.
The conversion step, while a barrier to adoption, reflects the same transition neuroimaging underwent from proprietary scanner formats to NIfTI: without a common data format, reusable tools cannot be built across laboratories.
% TODO: quantify format counts if possible (Neo lists 36 RawIO modules as of 2026-03; NeuroConv docs claim ~50)
Conversion tools such as Neo and NeuroConv support reading from dozens of acquisition formats and lower this barrier in practice.

The major neurophysiology repositories already operate on principles that microephys-BIDS complements.
DANDI requires NWB and supports BIDS validation for uploaded datasets.
EBRAINS (\rrid{RRID:SCR\_019260}) recommends BIDS for human neuroimaging data and provides general data organisation guidelines (modality separation, subject-based hierarchy, machine-readable metadata) that align with BIDS conventions; its recommendation has not yet extended to electrophysiology because no animal microelectrode extension existed until now.
Repositories such as GIN (\rrid{RRID:SCR\_015864}) store data with full filesystem fidelity, making the benefits of a harmonized directory structure immediately apparent to both humans and machines.

Adoption of this extension would enable BIDS-Apps that exploit the standardized structure and metadata for automated processing workflows, and provide laboratories with a consistent organisational framework.
Domain-specific standards address the unique requirements of individual modalities, but modern neurophysiology experiments increasingly combine electrophysiology with neuroimaging, optical physiology, and behavioral recordings in the same subjects and sessions.
BIDS complements these standards by providing a modality-spanning organisational layer, and with the addition of microelectrode electrophysiology, now covers the full range of modalities common in systems neuroscience laboratories.

\section*{Code Availability}

Using NWB and NIX as the underlying data format allows microephys BIDS data to utilize the sophisticated software ecosystems associated with these data standards.

The NWB software ecosystem contains a suite of tools to convert data from the various common electrophysiology data formats to NWB.
PyNWB (\rrid{RRID:SCR\_017452}) and MatNWB (\rrid{RRID:SCR\_021156}) provide flexible APIs for writing NWB files~\citep{rubel_neurodata_2022}.
NeuroConv automates conversion from 60+ formats, guiding the user in best practices~\citep{mayorquin_neuroconv_2025}.
NWB Inspector (\rrid{RRID:SCR\_025465}) provides automated checks to validate NWB files against best practices, and NWB GUIDE provides a graphical interface that provides the features of NeuroConv and NWB Inspector to non-programmers.

30+ popular analysis tools support NWB as an input and/or output format, many of which are relevant to microephys data.
SpikeInterface provides tools for creating spike sorting pipelines~\citep{buccino_spikeinterface_2020}.
Neurosift (\rrid{RRID:SCR\_025466}) provides interactive dashboards for visualizing NWB data~\citep{magland_neurosift_2024}, and Pynapple (\rrid{RRID:SCR\_023312})~\citep{viejo_pynapple_2023} provides tools for common neurophysiology analyses.

NIX can be written from NEO~\citep{garcia_neo_2014}, which supports reading diverse electrophysiology data formats.
NIX also supports data analysis libraries, such as Elephant (\rrid{RRID:SCR\_003833})~\citep{denker_collaborative_2018}.
